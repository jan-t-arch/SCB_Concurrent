Aggregate Premium Maximum Check
總保費上限檢查

本頁解釋當客戶同日投保多份保單時,系統如何加總所有保單保費(轉 HKD)再與上限比較,判斷是否超標。分三種情況(單筆 / 每月 / 混合),每種計法不同。選擇 scenario、加入保單、修改數值,即可查看實時逐步計算。

⚠ 最易混淆之處 / The counter-intuitive part

保單清單 / Policies

供款能力上限 / Max premium (affordability)

依已確認公式(見下)即場計出上限,直接餵入右方 verdict。cappedDI% / cappedA% = 用戶 range 取 Max 後,再按客戶標籤 cap 至 30/50。

逐步計算 / Step-by-step

✓ 已確認 / Confirmed

  • 所有金額轉為 HKD 後才加總;當前申請自身保費計入;手動 / 系統搜尋 / 當前申請同屬一客戶(client-level 檢查)。
  • 嚴格小於 strict <:總數 < 上限 = Pass;等於上限 = Fail(唔係 pass)。
  • 三種 case(banner):
    A 單筆  Σ single < maxSinglePremium
    B 每月  Σ monthly < maxMonthlyPremium(保持月供)
    C 混合  Σ single + Σ(monthly × pt × 12) < maxSinglePremium(全部換算 one-off,pt = 整組最大)
  • B vs C 差異屬有意設計(OQ2 已 close): B 把所有嘢拉成每月比 maxMonthly;C 把所有嘢拉成一筆過比 maxSingle。同一張月費保單兩者計法不同係正常。
  • 上限公式(Case A / B 圖):
    maxSingle = ndi≤0 ? (nla − mog×12×pt)×capA% : nla×capA%
    maxMonthly = ndi≤0 ? ((nla − mog×12×pt)×capA%)/pt/12 : (ndi×capDI% + nla×capA%)/pt/12
    結果四捨五入、clamp 至 0;single 於 nla<0 時回 0。mixed 之上限不計 disposable income(只用資產)。
